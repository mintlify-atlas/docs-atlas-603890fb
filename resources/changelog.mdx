## Recent Changes

### Breaking Changes

The latest version includes important breaking changes:

<Warning>
  If you're upgrading from an older version, please review the [Migration Guide](/resources/migration-guide) for detailed upgrade instructions.
</Warning>

<AccordionGroup>
  <Accordion title="Hints are Normalized by Default">
    `createHints(...)` now uses Arabic-first normalization for matching and mining. If you relied on exact string matching, update your expectations and/or pass explicit normalization options.
    
    **Reference**: [src/utils/textUtils.ts:121-156](https://github.com/ragaeeb/paragrafs/blob/main/src/utils/textUtils.ts#L121-L156)
  </Accordion>
  
  <Accordion title="ALWAYS_BREAK is a True Hard Boundary">
    Segments or lines after an `ALWAYS_BREAK` must not be merged into previous segments. This creates an absolute boundary in segment processing.
    
    **Reference**: 
    - [src/transcript.ts:95-167](https://github.com/ragaeeb/paragrafs/blob/main/src/transcript.ts#L95-L167)
    - [src/transcript.ts:173-211](https://github.com/ragaeeb/paragrafs/blob/main/src/transcript.ts#L173-L211)
  </Accordion>
</AccordionGroup>

## Semantic Versioning

Paragrafs follows [Semantic Versioning](https://semver.org/) (semver):

- **Major version** (X.0.0) - Breaking changes that require migration
- **Minor version** (0.X.0) - New features, backwards compatible
- **Patch version** (0.0.X) - Bug fixes, backwards compatible

<Tip>
  Pin to specific versions in your `package.json` to avoid unexpected breaking changes:
  ```json
  {
    "dependencies": {
      "paragrafs": "1.6.0"
    }
  }
  ```
</Tip>

## Contributing to Releases

If you're contributing to Paragrafs:

- Follow [conventional commit](https://www.conventionalcommits.org/) format
- Use `feat:` for new features (minor version bump)
- Use `fix:` for bug fixes (patch version bump)
- Use `BREAKING CHANGE:` in commit body for breaking changes (major version bump)

Learn more in the [Contributing Guide](/resources/contributing).
